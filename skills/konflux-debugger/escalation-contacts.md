# Escalation Contacts Quick Reference

**For platform-level incidents requiring a WebRCA: `#wg-sre-konflux`**

For team-specific escalation, use the Slack handles in the contacts table below.

---

## When to Escalate vs Keep Digging

| Escalate | Keep Digging |
|----------|-------------|
| Platform-level outage confirmed across clusters | Single-tenant issue with clear error messages |
| Issue requires write access to resolve | Known failure pattern with documented resolution |
| Issue spans multiple services/teams | User-side config error (Dockerfile, component config) |
| No response from initial investigation | Issue can be diagnosed with read-only commands |
| Exodus-gw, IIB, or Pyxis production outage | Single pipeline failure with identifiable root cause |

## Standard Incident Pattern

For **ALL** external and internal service incidents:

1. Inform stakeholders in `#announce-konflux-incidents`
2. Identify impact on Konflux
3. Follow up until resolution
4. Document in WebRCA if applicable
